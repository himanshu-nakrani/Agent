# Prototype Instructions

Run the local server yourself and open the preview in the browser available to this environment. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Learner reading preferences

- Deep-reading views must not retain a tall, empty module-navigation column.
- Use a full-width, centered reader layout with a restrained line length on large displays.
- Long-form learning text should prioritize contrast, generous line height, and readable body type over decorative display styling.

## Information architecture

- Major learning surfaces should be dedicated, directly addressable pages rather than sections of one long scroll.
- Keep Home, curriculum, individual chapters, framework guides, labs, and papers distinct while preserving the calm editorial system between routes.
